# EdgeOne 规则配置匹配类型修复指南

## 📋 问题分析

根据您提供的截图，我发现了配置错误：

**当前配置**：
- **匹配类型**：客户端 IP
- **匹配内容**：`/auth/*`（路径）
- **错误提示**：`请输入正确的IP或IP段`

**问题原因**：
- 您选择了 IP 匹配类型，但输入了 URL 路径
- IP 匹配类型只能匹配 IP 地址或 IP 段，不能匹配 URL 路径

## 🛠️ 解决方案

您需要选择正确的匹配类型来匹配 URL 路径。以下是正确的配置步骤：

### 步骤 1：选择正确的匹配类型

在 EdgeOne 控制台中，您需要选择以下匹配类型之一：

#### 选项 1：URL 路径匹配（推荐）
- **匹配类型**：选择 **"URL 路径"** 或 **"请求路径"**
- **匹配内容**：`/auth/*`
- **说明**：专门用于匹配 URL 路径

#### 选项 2：请求头匹配
- **匹配类型**：选择 **"请求头"**
- **匹配内容**：`/auth/*`
- **说明**：通过请求头匹配路径（较少使用）

#### 选项 3：请求方法匹配
- **匹配类型**：选择 **"请求方法"**
- **匹配内容**：`GET`
- **说明**：匹配 HTTP 方法（需要结合其他规则）

### 步骤 2：正确的配置示例

#### 示例 1：使用 URL 路径匹配（推荐）

| 字段 | 值 | 说明 |
|------|-----|------|
| **域名** | `lewis-testcase-platform-xyqvs7bh.edgeone.cool` | 项目域名 |
| **规则名称** | `Block GET requests to auth endpoints` | 规则名称 |
| **匹配类型** | `URL 路径` | 正确的匹配类型 |
| **匹配方式** | `匹配` | 匹配方式 |
| **匹配内容** | `/auth/*` | URL 路径模式 |
| **执行动作** | `拦截` | 拦截动作 |
| **状态** | `开启` | 规则状态 |

#### 示例 2：使用请求方法匹配

| 字段 | 值 | 说明 |
|------|-----|------|
| **域名** | `lewis-testcase-platform-xyqvs7bh.edgeone.cool` | 项目域名 |
| **规则名称** | `Block GET requests to auth endpoints` | 规则名称 |
| **匹配类型** | `请求方法` | 匹配 HTTP 方法 |
| **匹配方式** | `匹配` | 匹配方式 |
| **匹配内容** | `GET` | HTTP 方法 |
| **执行动作** | `拦截` | 拦截动作 |
| **状态** | `开启` | 规则状态 |

### 步骤 3：配置所有规则

#### 规则 1：阻止 GET 请求到认证接口
- **匹配类型**：URL 路径
- **匹配内容**：`/auth/*`
- **执行动作**：拦截

#### 规则 2：允许 POST 请求到登录接口
- **匹配类型**：URL 路径
- **匹配内容**：`/auth/login`
- **执行动作**：允许

#### 规则 3：允许 POST 请求到注册接口
- **匹配类型**：URL 路径
- **匹配内容**：`/auth/register`
- **执行动作**：允许

#### 规则 4：允许认证接口的其他方法
- **匹配类型**：URL 路径
- **匹配内容**：`/auth/*`
- **执行动作**：允许

## 🔧 故障排除

如果仍然遇到问题：

1. **确认匹配类型**：确保选择了 URL 路径匹配
2. **检查路径格式**：确保路径以 `/` 开头，如 `/auth/*`
3. **确认域名**：确保选择了正确的域名
4. **等待生效**：配置可能需要 5-10 分钟生效

## 📝 关键要点

- **不要使用 IP 匹配类型**来匹配 URL 路径
- **使用 URL 路径匹配类型**来匹配 `/auth/*` 路径
- **确保路径格式正确**：以 `/` 开头，使用 `*` 作为通配符

按照这些步骤操作，您的 EdgeOne 配置将正确解决登录接口 GET 请求的安全问题。
