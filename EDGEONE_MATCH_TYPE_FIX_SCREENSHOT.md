# EdgeOne 规则配置匹配类型修复指南（基于截图）

## 📋 问题分析

根据您提供的截图，我发现了配置错误：

**当前配置**：
- **匹配类型**：客户端 IP
- **匹配内容**：`/auth/*`（路径）
- **错误提示**：`请输入正确的IP或IP段`

**问题原因**：
- 您选择了 IP 匹配类型，但输入了 URL 路径
- IP 匹配类型只能匹配 IP 地址或 IP 段，不能匹配 URL 路径

## 🖱️ 具体修复步骤（基于您的截图）

### 步骤 1：取消当前配置

在截图中的 **"取消"** 按钮上点击，取消当前错误的配置。

![取消按钮](assets/image-c895dcb7-5239-4e3e-bc6b-43d8d7ce0ed0.png)

### 步骤 2：重新添加规则

点击 **"添加规则"** 按钮，重新开始配置。

### 步骤 3：选择正确的匹配类型

在新的配置对话框中，找到 **"匹配类型"** 下拉菜单，选择 **"URL 路径"** 或 **"请求路径"**：

| 字段 | 值 | 说明 |
|------|-----|------|
| **域名** | `<your-frontend-domain>` | 项目域名 |
| **规则名称** | `Block GET requests to auth endpoints` | 规则名称 |
| **匹配类型** | `URL 路径` | 正确的匹配类型（替换客户端 IP） |
| **匹配方式** | `匹配` | 匹配方式 |
| **匹配内容** | `/auth/*` | URL 路径模式（保持不变） |
| **执行动作** | `拦截` | 拦截动作 |
| **状态** | `开启` | 规则状态 |

### 步骤 4：保存配置

点击 **"保存"** 按钮，保存正确的配置。

### 步骤 5：配置其他规则

重复步骤 2-4，配置其他规则：

#### 规则 2：允许 POST 请求到登录接口
- **匹配类型**：URL 路径
- **匹配内容**：`/auth/login`
- **执行动作**：允许

#### 规则 3：允许 POST 请求到注册接口
- **匹配类型**：URL 路径
- **匹配内容**：`/auth/register`
- **执行动作**：允许

#### 规则 4：允许认证接口的其他方法
- **匹配类型**：URL 路径
- **匹配内容**：`/auth/*`
- **执行动作**：允许

## 📊 预期结果

配置完成后，规则列表应该显示：

| 规则名称 | 状态 | 操作 |
|---------|------|------|
| Block GET requests to auth endpoints | 开启 | 编辑/删除 |
| Allow only POST to login | 开启 | 编辑/删除 |
| Allow only POST to register | 开启 | 编辑/删除 |
| Allow auth API methods | 开启 | 编辑/删除 |

## 🧪 验证配置

使用以下命令验证配置：

```bash
# 测试 GET 请求被拦截
curl -X GET "https://<your-frontend-domain>/api/auth/login" -v

# 测试 POST 请求被允许
curl -X POST "https://<your-frontend-domain>/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.invalid","password":"<占位-勿用真实口令>"}' \
  -v
```

## 🔧 关键修复点

1. **匹配类型选择**：从 "客户端 IP" 改为 "URL 路径"
2. **路径格式**：确保路径以 `/` 开头，如 `/auth/*`
3. **域名确认**：确保选择了正确的域名
4. **状态开启**：确保规则状态为开启

## 📝 注意事项

- **不要使用 IP 匹配类型**来匹配 URL 路径
- **使用 URL 路径匹配类型**来匹配 `/auth/*` 路径
- **确保路径格式正确**：以 `/` 开头，使用 `*` 作为通配符

按照这些步骤操作，您的 EdgeOne 配置将正确解决登录接口 GET 请求的安全问题。如果仍有问题，请检查匹配类型是否正确选择。
