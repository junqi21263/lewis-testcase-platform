# AI 测试用例生成平台自测计划

## 📋 自测目标

确保前后端功能正常，包括：
- 用户注册和登录
- 密码重置和邮箱验证
- 用户资料管理
- 安全特性（密码强度、速率限制等）

## 🧪 测试场景

### 1. 用户注册流程
- 场景 1.1：正常注册（邮箱可用，密码符合要求）
- 场景 1.2：邮箱已存在（冲突异常）
- 场景 1.3：用户名已存在（冲突异常）
- 场景 1.4：密码强度不足（验证失败）
- 场景 1.5：邮箱格式不正确（验证失败）

### 2. 用户登录流程
- 场景 2.1：正常登录（邮箱和密码正确）
- 场景 2.2：邮箱不存在（未找到异常）
- 场景 2.3：密码错误（未找到异常）
- 场景 2.4：密码强度不足（验证失败）
- 场景 2.5：邮箱格式不正确（验证失败）

### 3. 密码重置流程
- 场景 3.1：发送重置链接（邮箱存在）
- 场景 3.2：发送重置链接（邮箱不存在）
- 场景 3.3：重置密码（令牌有效）
- 场景 3.4：重置密码（令牌已过期）
- 场景 3.5：重置密码（新密码强度不足）

### 4. 用户资料管理
- 场景 4.1：更新用户名（新用户名可用）
- 场景 4.2：更新用户名（新用户名已存在）
- 场景 4.3：更新头像
- 场景 4.4：修改密码（旧密码正确）
- 场景 4.5：修改密码（旧密码错误）

### 5. 安全特性测试
- 场景 5.1：暴力破解测试（多次失败登录）
- 场景 5.2：密码强度检查（不同强度密码）
- 场景 5.3：速率限制测试（快速连续请求）
- 场景 5.4：安全头检查（XSS防护等）

### 6. 用户体验测试
- 场景 6.1：表单验证反馈
- 场景 6.2：加载状态显示
- 场景 6.3：错误信息显示
- 场景 6.4：成功提示显示

## 📝 测试步骤

### 步骤 1：环境准备
1. 启动后端服务
2. 启动前端服务
3. 清空数据库（如果需要）

### 步骤 2：执行测试
1. 按照测试场景逐一执行
2. 记录每个场景的结果
3. 记录任何错误或异常

### 步骤 3：结果分析
1. 分析测试结果
2. 记录所有问题
3. 修复发现的问题

### 步骤 4：回归测试
1. 修复问题后重新测试
2. 确认问题已解决
3. 确认没有引入新问题

## 📁 文件分类建议

### 1. 后端目录
```
backend/
├── src/
│   ├── common/          # 公共模块
│   │   ├── config/      # 配置文件
│   │   ├── decorators/  # 装饰器
│   │   ├── guards/      # 守卫
│   │   ├── middleware/  # 中间件
│   │   ├── validators/  # 验证器
│   │   └── services/    # 服务
│   ├── modules/         # 业务模块
│   │   ├── auth/        # 认证模块
│   │   ├── files/       # 文件模块
│   │   ├── ai/          # AI模块
│   │   ├── testcases/   # 用例模块
│   │   ├── templates/   # 模板模块
│   │   ├── teams/       # 团队模块

│   ├── prisma/          # 数据库
```
