# EdgeOne 路由规则配置步骤（基于截图指导）

## 📋 当前状态分析

根据您提供的截图，您已经：
1. 进入了 EdgeOne 控制台的 **"自定义规则"** 页面
2. 看到了 **"添加规则"** 按钮
3. 准备开始配置路由规则

## 🖱️ 具体操作步骤（基于截图）

### 步骤 1：点击 "添加规则" 按钮

在截图中的 **"添加规则"** 按钮上点击，打开规则配置对话框。

![添加规则按钮](assets/image-135add63-0c11-42a5-89a8-e09fd632f68f.png)

### 步骤 2：配置第一个规则 - 阻止 GET 请求到认证接口

在弹出的对话框中填写以下信息：

| 字段 | 值 | 说明 |
|------|-----|------|
| **域名** | `<your-frontend-domain>` | 选择您的项目域名 |
| **规则名称** | `Block GET requests to auth endpoints` | 清晰的规则名称 |
| **匹配条件** | `客户端 IP` | 选择匹配条件类型 |
| **匹配类型** | `客户端 IP` | 确保选择正确的匹配类型 |
| **匹配方式** | `匹配` | 选择匹配方式 |
| **匹配内容** | `/auth/*` | 路径匹配模式，匹配所有认证接口 |
| **执行动作** | `拦截` | 选择拦截动作 |
| **状态** | `开启` | 确保开关是开启状态 |

![规则配置对话框](assets/image-25b01a77-ce05-4684-b36d-f79457498009.png)

### 步骤 3：配置第二个规则 - 允许 POST 请求到登录接口

| **匹配条件** | `客户端 IP` | 同上 |
| **匹配类型** | `客户端 IP` | 同上 |
| **匹配方式** | `匹配` | 同上 |
| **匹配内容** | `/auth/login` | 仅匹配登录接口 |
| **执行动作** | `允许` | 选择允许动作 |
| **状态** | `开启` | 确保开关是开启状态 |

## 📊 规则列表预期结果

配置完成后，规则列表应该显示：

| 规则名称 | 状态 | 操作 |
|---------|------|------|
| Block GET requests to auth endpoints | 开启 | 编辑/删除 |
| Allow only POST to login | 开启 | 编辑/删除 |
| Allow only POST to register | 开启 | 编辑/删除 |
| Allow auth API methods | 开启 | 编辑/删除 |

## 🔧 高级配置选项

### 1. 优先级调整
如果需要手动调整优先级：
1. 找到规则列表中的 **"操作"** 列
2. 点击 **"编辑"** 调整优先级
3. 确保拦截规则优先级高于允许规则

### 2. 规则测试
EdgeOne 提供规则测试功能：
1. 选择要测试的规则
2. 输入测试 URL
3. 模拟请求并查看结果

## 🧪 配置验证

### 1. 等待配置生效
- 配置完成后，等待 5-10 分钟让规则传播到所有边缘节点
- EdgeOne 通常会显示配置生效时间

### 2. 使用验证脚本测试
运行之前创建的验证脚本：
```bash
./verify-edgeone-config.sh
```

### 3. 手动测试
使用 curl 命令测试：

```bash
# 测试 GET 请求被拦截
curl -X GET "https://<your-frontend-domain>/api/auth/login" -v

# 测试 POST 请求被允许
curl -X POST "https://<your-frontend-domain>/api/auth/login" \
  -H "Content-Type: application/json" \
  -d '{"email":"user@example.invalid","password":"<占位-勿用真实口令>"}' \
  -v
```

## 📝 注意事项

1. **路径匹配**：确保路径格式正确，使用 `/auth/*` 而不是 `auth/*`
2. **方法匹配**：某些版本可能需要单独配置 HTTP 方法
3. **域名选择**：确保选择了正确的域名
4. **状态检查**：定期检查规则状态是否为开启
